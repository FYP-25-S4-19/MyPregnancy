1. Introduction  
 
The objective of this test plan is to verify that Registered Doctor can successfully access, 
manage, and interact with the application features. This includes authentication, profile 
management, educational content management, community participation, and consultation 
request handling. 
 
2. Scope 
 
Authentication: Login, Logout 
Account Management: View, Update, Delete account 
Education Content: View, Create, Share, Save 
Community Forum: View threads, Create/Edit/Delete threads, Create/Edit/Delete comments 
Consultation Requests: View, Accept, Reject 
 
3. Test Environment 
a. Hardware & platform  
● Mobile: Testing via Mobile emulator (simulating Android) 
b. Software & Browser 
● Microsoft Edge 
c. Test Data setup 
● Dummy Doctor Account 
● Dummy Educational Content 
● Dummy Community Threads 
● Dummy Consultation Requests 
● Dummy Comments 
 
4. Test Strategy 
a. Methodology 
■ Manual testing for all test cases 
■ Functional validation of Doctor workflows 
■ UI and usability verification for Doctor features 
b. Defect Management 
■ Bug Tracking Tools: Github (logging)
